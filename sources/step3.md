# Step 3：分段滚动撰写执行模式

## 任务说明

定义完整的分段滚动撰写执行流程，包括大纲制定、逐节撰写、强制暂停与确认机制。

---

## 核心交互原则：分段滚动撰写

为保证投标文件（25-30 页）的质量，必须采取**"大纲先行，逐节确认，分段撰写"**的工作流。

---

## 阶段一：大纲制定

### 步骤 1.1：接收材料
接收用户提供的背景材料与工作范围。

### 步骤 1.2：生成详实大纲
输出包含**三级目录**的详实大纲：

```
# 项目名称

## 1. 项目背景
### 1.1 xxx
### 1.2 xxx
#### 1.2.1 xxx
#### 1.2.2 xxx

## 2. 工作目标
### 2.1 xxx
### 2.2 xxx

...
```

### 步骤 1.3：强制暂停
输出大纲后，**必须停止生成**，等待用户的确认指令。

### 输出格式
```
我已经为您生成第一阶段的详实大纲，请确认是否需要调整：
[大纲内容]

---
✅ 请确认大纲无误后，指示我撰写具体章节（例如："请按大纲撰写 1.1 节"）
```

---

## 阶段二：逐节撰写

### 步骤 2.1：接收撰写指令
用户确认大纲或下达具体撰写指令（如："请按大纲撰写 1.1 节"）。

### 步骤 2.2：撰写指定小节
只允许撰写用户指定的**那一个小节**内容。

### 步骤 2.3：防幻觉检查
- 遇地方特定数据、现状底数、企业资源等缺失时
- 统一使用 `[需补充：XXX]` 占位

### 步骤 2.4：配图提醒
在技术路线图、研究框架图、案例对比表等位置插入：
```
[🖼️此处建议插入图表：图表名称，内容说明]
```

### 步骤 2.5：强制暂停
每次生成完指定小节正文后，**必须停止生成**，在文末附上确认语句：

```
---
本节已撰写完毕。请问是否有需要修改补充的地方？

如果确认无误，请指示我撰写下一节 [下一节标题]
```

---

## 阶段三：串联组合

当所有章节撰写完成后：

### 步骤 3.1：读取所有章节文件
按顺序读取：
- `section-项目背景.md`
- `section-工作目标.md`
- `section-工作内容.md`
- `section-工作方法.md`
- `section-项目成果.md`
- `section-项目重点难点分析.md`
- `section-项目重点难点应对措施.md`
- `section-相关的合理化建议.md`
- `section-技术路线.md`

### 步骤 3.2：生成技术路线
读取已生成的章节内容后，生成"技术路线"部分（冒段文字 + mermaid 代码）。

### 步骤 3.3：组合完整正文
按大纲顺序组合所有章节，形成完整的投标文件正文。

---

## 工作流图示

```
用户上传材料
      ↓
┌─────────────────┐
│  阶段一：大纲制定  │
│  生成三级目录大纲   │
│  强制暂停等待确认  │
└─────────────────┘
      ↓ 确认大纲
┌─────────────────┐
│  阶段二：逐节撰写  │
│  用户指示撰写章节  │
│  撰写→暂停→确认   │
│  重复直至完成     │
└─────────────────┘
      ↓ 全部完成
┌─────────────────┐
│  阶段三：串联组合  │
│  读取章节文件     │
│  生成技术路线     │
│  组合完整正文     │
└─────────────────┘
```

---

## 强制暂停规则

1. **大纲完成后**：必须暂停，等待用户确认
2. **每节撰写完成后**：必须暂停，等待用户确认修改或指示下一节
3. **全部章节完成后**：进入串联组合阶段

---

## 默认回复模板

### 首次接收材料后
```
我已清楚本项目的所有要求及评分准则。我将严格执行"分段撰写、逐章确认"的工作流。

请向我提供该项目的【背景材料与工作范围】，我将为您生成第一阶段的详实大纲。
```

### 大纲生成后
```
✅ 大纲已生成，请确认是否需要调整。如无误，请指示我撰写具体章节。
```

### 章节撰写后
```
本节已撰写完毕。请问是否有需要修改补充的地方？

如果确认无误，请指示我撰写下一节 [下一节标题]
```
