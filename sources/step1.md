# Step 1：旧 Prompt 适配工作流

## 任务说明

本 step 定义了从"用户提供旧 Prompt 模板"到"生成适配新项目的完整 Prompt"的完整工作流。

---

## 输入材料

### 输入 A：旧 Prompt 模板（固定）
- 文件名：`old-prompt.md`
- 内容：已成熟的 Prompt 框架，包含角色定义、背景上下文、篇幅要求、分段滚动撰写机制、防幻觉与占位机制、配图提醒机制、评分准则约束等

### 输入 B：新招标需求文件 / 项目材料（用户提供）
- 招标文件、采购需求书、评分标准、项目任务书等

---

## 执行步骤

### Step 1.1：接收材料

等待用户提供：
1. 确认使用 `old-prompt.md` 作为旧 Prompt 模板
2. 提供新的招标需求文件或项目材料

### Step 1.2：提取新项目关键信息

从新招标需求文件中提取并归纳：

| 序号 | 信息项 |
|------|--------|
| 1 | 项目名称 |
| 2 | 项目性质与专业类型 |
| 3 | 项目目标 |
| 4 | 服务/研究对象 |
| 5 | 工作范围 |
| 6 | 核心工作内容 |
| 7 | 技术路线关键词 |
| 8 | 项目成果要求 |
| 9 | 篇幅或编制深度要求 |
| 10 | 评分标准及得分重点 |
| 11 | 特殊表达要求 |
| 12 | 需要重点检索的政策方向 |
| 13 | 需要重点借鉴的案例方向 |
| 14 | 与旧 Prompt 明显冲突的内容 |
| 15 | 可以直接沿用的内容 |

### Step 1.3：差异识别

明确判断：
- **保留项**：哪些段落原样保留
- **替换项**：哪些段落局部替换（如项目名称、领域、背景等）
- **重写项**：哪些段落整体重写
- **新增项**：哪些新要求需要新增
- **删除项**：哪些旧内容需要删除

### Step 1.4：输出适配结果

#### A. 适配结论摘要
用简洁语言说明：
- 旧 Prompt 的哪些核心结构被保留
- 哪些项目化内容被替换
- 哪些内容被新增
- 哪些内容被删除或弱化
- 本次改写的总体逻辑

#### B. 旧 Prompt 适配诊断
按四类列出关键变化：

**1. 保留项**
- 列表项...

**2. 替换项**
- 列表项...

**3. 新增项**
- 列表项...

**4. 删除项**
- 列表项...

#### C. 完整新 Prompt
输出一版完整的、可以直接复制使用的新 Prompt：
- 结构完整
- 保留旧 Prompt 的成熟框架
- 已吸收新项目关键信息
- 适合作为下一轮正式写作的执行 Prompt

#### D. 关键替换点清单
清单形式，示例：
- 项目名称：由 `[旧项目名称]` 改为 `[新项目名称]`
- 专业角色：由 `XXX` 改为 `XXX`
- 内容导向：由 `XXX` 改为 `XXX`
- 技术路线关键词：由 `XXX` 改为 `XXX`
- 成果体系：由 `XXX` 改为 `XXX`
- 政策搜索方向：由 `XXX` 改为 `XXX`
- 案例借鉴方向：由 `XXX` 改为 `XXX`
- 配图提醒重点：由 `XXX` 改为 `XXX`
- 评分约束重点：新增/替换 `XXX`

---

## 核心原则

1. **保留优先**：除非旧 Prompt 与新项目明显冲突，否则优先保留原有结构与工作机制
2. **定向替换**：重点替换项目名称、类型、领域、背景、目标、成果、评分准则等
3. **不得机械替换**：需根据新项目判断哪些原样保留、哪些局部替换、哪些整段重写
4. **不得擅自补造**：未明确的信息统一使用 `[需补充：XXX]` 占位
5. **工作流尽量不变**：大纲先行、逐节确认、分段撰写等成熟机制必须保留
6. **评分准则强绑定**：严格对照评分标准写作

---

## 优先级规则

当旧 Prompt 与新招标需求文件冲突时：
1. 新招标文件原文要求
2. 新项目评分标准与采购需求
3. 旧 Prompt 中可复用的写作机制
4. 旧 Prompt 中与旧项目绑定的专业表述

---

## 硬性限制

1. 任务只是"改写 Prompt"，不是开始撰写投标正文
2. 不能跳过对比分析直接另起炉灶
3. 不能破坏旧 Prompt 的成熟结构
4. 不能编造招标文件没有的项目细节
5. 遇到信息不足时不得自行脑补，必须使用 `[需补充：XXX]` 占位
6. 改写后的新 Prompt 必须仍然适合"分段滚动撰写"工作流

---

## 默认行为

接收到用户确认后，立即进入"Prompt 适配模式"，直接输出适配结果。除非用户明确要求，否则不反问"是否开始"。

---

## 输出文件

将"完整新 Prompt"内容保存为 `new-prompt.md`
